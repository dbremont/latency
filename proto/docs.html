Framework Goal

This document defines a comprehensive framework for modeling, analyzing, and representing system latency. The goal is to construct a data-driven, explainable, and multi-resolution latency model that captures the behavior of requests across system routes, workloads, and resources.

Data Quality: Right-Censoring

The dataset exhibits right-censoring bias.

  • If a user cancels a request (due to latency), the request is not fully observed.
  • High-latency events are systematically underrepresented.
  • The observed distribution is truncated on the upper tail.
Consequences: Underestimation of mean/variance, distortion of tail quantiles (p95, p99), misleading worst-case analysis.

System Definition

We define the system as a tuple of spaces and operators.

M = (H, Z, E, W, T, C, P)
H
History — sequences of past trajectories
Z
Latent Space — hidden internal factors (congestion, unserved work)
E
Environment — exogenous conditions
W
Workload Space — intensity, concurrency, pressure

History Process

Ht ∈ H
Ht = {(Sτ, Zτ, Eτ, Wτ) : τ ≤ t}

(Ht) is the full system memory (grows with time).

State Construction

St = φ(Ht)

φ: H → S is the history projection operator. St is a lossy compression of Ht.

Dynamics & Intrinsic Cost

Dynamics
St+1 = T(Ht, Zt, Et, Wt)

The state is a sufficient statistic of all past evolution.

Intrinsic Cost (Latency)
Lt = C(St, St+1, Zt, Et, Wt)

Lt ∈ ℝ+. Represents the cost of transition under constraints.

Note: The system state is fully determined by its history and the environment. For implementation, an explicit state space may be introduced, but is not required for specification. A primary analysis strategy is simulation via generation of histories.

Metric Evaluation Criteria

Operational

  • • Cold-Start Robustness: Valid outputs with insufficient history.
  • • Sensitivity to Spikes: Reflect extreme single requests.
  • • Sensitivity to Drift: React to gradual increases.
  • • Noise Robustness: Ignore normal small jitter.

Structural

  • • Composability: Aggregation across routes, resampling.
  • • Interpretability: Maps to behavior, monotonic health relationship.

Validation Strategy

How to validate the resulting representations?

Consistency Checks

Verify that aggregated metrics (e.g., P95 over 1 hour) match those derived from finer resolutions (1-minute buckets).

Simulation Fidelity

Compare generated histories and evolutions against observed system behavior to validate the system model dynamics.

Operational Testing

Evaluate metrics against cold-start robustness, noise sensitivity, and composability requirements.
